**Serious bug found and fixed, worth flagging to any agent or human working here next**: during this pass, five pages — `index.html`, `contact.html`, `features.html`, `pricing.html`, `training.html` — were discovered to be genuinely truncated on disk, cut off mid footer with the closing `</footer>`, sticky CTA, script tag, `</body>` and `</html>` missing entirely. This was confirmed at the byte level (`wc -c` and `tail -c` agreeing with the Read tool), not a caching illusion. Root cause is most likely a bulk Node/Python script reading and rewriting these files through this sandbox's Linux mount of the Windows folder shortly after they had been edited through the Windows side file tools; the mount's read can lag behind the true file state, and a script that reads, modifies, and writes the whole file back out will happily commit that stale, truncated read as the new permanent content. All five files have been fully reconstructed and each one individually re-verified via the Read tool (not the shell, which continued to report stale byte counts for these files long after the fix was written). **Recommendation for future work in this repository: avoid bulk shell scripts that read and rewrite whole HTML files through this sandbox mount. Prefer the Edit tool for targeted changes, and if a bulk script is unavoidable, re-verify every touched file's tail through the Read tool afterward, not through the shell.**
