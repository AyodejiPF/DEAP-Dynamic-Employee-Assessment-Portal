# Product Backlog Discipline

Use this skill to convert scattered product work into an ordered backlog that can be executed without losing context.

## Workflow

1. Identify the product, business goal, current user, and live risk.
2. Separate the request into feature, bug, design, content, data, launch, and operations items.
3. Merge duplicates and preserve the strongest wording from the clearest source.
4. Mark each item as urgent, high, medium, low, blocked, or archive.
5. Add a short reason for priority using impact, risk, revenue value, customer pain, and implementation effort.
6. Define done criteria for every active item.
7. Identify dependencies, approval gates, data safety concerns, and deployment needs.
8. Produce a backlog table with owner, next action, status, and evidence source.

## Priority Rules

Put revenue protection, data safety, live user impact, and blocking defects first.

Put visual polish, exploratory ideas, and optional enhancements after core workflow reliability.

Do not allow one broad request to become a vague mega task. Break it down until each item has one clear outcome.

## Output Format

Use a table with these columns:

Priority, item, type, evidence, why it matters, done criteria, next action, risk, status.

End with the top three actions to do now.
