# Technical Project Management

Use this skill when a project has many moving parts and needs a clear operating rhythm.

## Workflow

1. Identify the project and current objective.
2. List completed work, active work, blocked work, and unstarted work.
3. Identify the safest last point.
4. Confirm the next action and why it is next.
5. Track verification, deployment, user approval, and unresolved risk.
6. Keep destructive or production actions behind explicit approval.
7. Maintain a concise status table.

## Status Model

Use these statuses:

Not started, in progress, blocked, awaiting approval, ready for verification, verified, deployed, archived.

## Progress Reporting

When Ayodeji asks if the work is done, answer directly first.

Then say:

Current stage

Completed items

Remaining items

Safest next step

Risk or blocker

## Output Format

Use a project control table with columns for workstream, status, evidence, next action, owner, and risk.
