# UI UX Acceptance

Use this skill to turn taste and visual direction into checkable acceptance criteria.

## Workflow

1. Identify the intended audience and primary task.
2. Extract the style rules from examples, brand notes, screenshots, or user comments.
3. Define acceptance criteria before editing.
4. Check layout, hierarchy, colour, spacing, typography, imagery, interaction, accessibility, and mobile behaviour.
5. Verify with screenshots where possible.
6. Fix the smallest surface first when the style is uncertain.
7. Expand only after the first surface passes visual judgement.

## Checks

Ask whether the interface is understandable, usable, visually aligned, responsive, accessible, and emotionally appropriate for the product.

For mobile, check touch size, scrolling, zoom behaviour, menus, forms, sticky actions, safe areas, and text fit.

For style matching, check material, shadows, edges, texture, thickness, lighting, icon style, and repeated patterns.

## Output Format

Provide criteria, findings, fixes, verification, and remaining visual risks.
